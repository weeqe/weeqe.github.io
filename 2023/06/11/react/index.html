React

ReactElement

ReactElement 是一个纯粹的 JavaScript对象，它有type属性，表示元素的类型，可以是一个HTML的标签名，也可以是一个React组件，还有 props属性，表示元素的属性和元素信息等，例如：className,style,onClick等。还有children 属性，表示它的子元素，可能是一个文本节点或者一个/多个 ReactElement。在渲染过程中，React 会根据 ReactElement 的类型和属性来创建真实的 DOM 元素或者调用相应的组件类进行渲染。

在 React 应用中，我们通常通过 JSX 语法来创建 ReactElement。

ReactElement 的实现原理比较简单。在 JSX 语法中，React 会将标签转换为 ReactElement，然后将它们构建成一个树形结构，这个树形结构也被称为 React 的虚拟 DOM。在 React 应用中，当需要更新 UI 时，React 会比较新旧虚拟 DOM 的差异，然后将差异应用到真实的 DOM 上，从而实现高效的 UI 更新。

Fiber

什么是Fiber

Fiber 是React16引入的一种新的协调机制，它是React内部的一种数据结构，用来管理组件的更新流程。Fiber通过链表的方式表示组件树，并且用于在多个渲染帧之间分割工作，以实现增量渲染和优先级调度。

Fiber的特点

  • 虚拟Dom: Fiber节点表示组件树种的一个虚拟节点，保存了该节点的信息，如类型，属性，子节点，alternate 等
  • 工作单元：每个Fiber节点都代表了React的一个工作单元，React通过遍历Fiber树来执行这些工作单元，实现组件的渲染和更新
  • 协调和调度：Fiber使用一种协调算法，可以中断，暂停和恢复渲染过程，以便更好的响应用户交互和优化性能

为什么引入Fiber

Fiber的引入主要是为了解决 React 中的两个问题：递归调用和同步渲染。 在React15之前，组件的更新过程都是通过递归调用来实现的，这样会导致调用栈过深，而且无法中断和恢复，对于用户的体验影响很大。而在React16中，Fiber将组件的更新过程改为了循环调用，这样可以实现中断和恢复，同时也更加灵活可控。

具体来说，当组件需要更新时，React 会创建一个新的 Fiber 树，通过遍历新旧 Fiber 树的差异，将需要更新的节点打上标记，然后再根据标记来执行实际的更新操作。这种方式可以在不阻塞主线程的情况下，按优先级和时间分片的方式更新 UI，从而提高性能和用户体验。

ReactElement 和 Fiber 的区别

  1. 概念层级不同：ReactElement是描述组件的虚拟元素，而Fiber是实现React调度和渲染的数据结构
  2. 可变性不同：ReactElement是不可变的，一旦创建就不能更改。而Fiber是可以在运行时更改和调度的
  3. 目标不同：ReactElement的目标是描述组件的结构和渲染方式，而Fiber的目标是实现高效的渲染和协调机制
  4. 应用场景不同：ReactElement适用于静态的、不会频繁变化的组件结构描述；而Fiber适用于处理复杂、大型组件树以及实现增量渲染和优先级调度等场景。
